Questão 01
O BR Code é o padrão brasileiro de QR Code para o Pix. Qual norma internacional ele adapta?
O BR Code é a adaptação brasileira do EMV-QRCPS (Europay, Mastercard, Visa QR Code Payment Specification). No Brasil, seu único uso é o Pix.
Questão 02
No formato TLV do BR Code, o que representa o "L" (Length) de um campo?
O "L" é o comprimento real do valor, em decimal com zero-padding (sempre 2 chars). É o tamanho exato — sem padding no valor. Campo opcional vazio é omitido por inteiro.
Questão 03
Qual é o valor exato da GUI (Globally Unique Identifier) que identifica o Pix no BR Code?
O valor correto é br.gov.bcb.pix — é o identificador único que diferencia um QR Code Pix de outros tipos de QR Code de pagamento. Vai no subcampo ID=00 da faixa 26.
Questão 04
Qual é a principal diferença técnica entre um QR Code Pix estático e um dinâmico?
No dinâmico, o campo 25 da faixa 26 contém uma URL sem o prefixo https:// (o app pagador adiciona). Essa URL retorna um JSON (payload Cob ou CobV) com valor, devedor, txid etc. Os dados de negócio não ficam no QR.
Questão 05
Um e-commerce quer que cada pedido gere um QR Code único, que expire em 30 minutos e impeça pagamento duplicado. Qual tipo de QR Code Pix atende esses requisitos e por quê?
O QR Dinâmico (Cob) foi criado exatamente para esse caso: valor único por transação, campo expiracao em segundos e status CONCLUIDA que bloqueia novos pagamentos após o primeiro. O QR Estático não tem como expirar nem impedir pagamentos múltiplos.
Questão 06
Uma empresa quer que ao receber um Pix, o sistema ERP identifique automaticamente a qual pedido aquele pagamento pertence — sem intervenção humana. Qual recurso do Pix viabiliza essa conciliação automática?
O txid é o elo entre o QR Code e o pedido: o recebedor o gera ao criar a cobrança (ou delega ao PSP), e quando o Pix chega o webhook o traz de volta. Com txid → pedido mapeado, a conciliação é automática e sem ambiguidade.
Questão 07
O que é o DICT no ecossistema Pix?
DICT = Diretório de Identificadores de Contas Transacionais. Quando o app pagador lê um QR Code, consulta o DICT para mapear a chave → banco + agência + conta. É a fonte de verdade do sistema.
Questão 08
Qual é a principal utilidade prática do E2EID (End-to-End ID) do ponto de vista de produto e operações?
O E2EID é o identificador global e único de cada transação Pix, gerado pelo banco emissor. Com ele, qualquer PSP consegue localizar o pagamento exato no SPI, solicitar uma devolução (MED ou operacional) ou responder a auditorias do Banco Central. É o "número de protocolo" do Pix.
Questão 09
Na fórmula do valor final da CobV — Vf = Vo − Va − Vd + Vj + Vm — o que representa cada variável? Qual alternativa está CORRETA?
Va = Abatimento (diminui sempre se configurado), Vd = Desconto (só antes/no vencimento), Vj = Juros (acresce após vencimento), Vm = Multa (aplicada uma vez após vencimento).
Questão 10
Quais são os dois mecanismos de autenticação obrigatórios para o cliente usar a API Pix?
OAuth 2.0 (fluxo client_credentials) para o token de acesso + mTLS (mutual TLS) com certificado ICP-Brasil na camada de transporte. Ambos são obrigatórios simultaneamente em toda chamada.
Questão 11
Um campo opcional do BR Code não tem valor a informar. O que deve ser feito?
No TLV do BR Code, campo opcional vazio = campo inteiro omitido. Não existe "comprimento zero" ou padding. Isso mantém o payload compacto e o QR Code com menos módulos.
Questão 12
Qual é a principal garantia operacional do SPI (Sistema de Pagamentos Instantâneos) que o torna adequado para cobranças por QR Code em qualquer horário do dia?
O SPI opera ininterruptamente — 24h, 7 dias, 365 dias, sem janelas de manutenção. Isso é o que permite o Pix funcionar numa compra de madrugada de sábado ou num feriado nacional. A liquidação é individual (LBTR), em segundos, diferente do TED (lotes, dias úteis).
Questão 13
O PSP pode reenviar a mesma notificação de webhook para o servidor do recebedor. Qual regra de negócio o sistema deve implementar para evitar processar o mesmo pagamento duas vezes?
O endToEndId (E2EID) é único por transação. Armazenando os E2EIDs já processados, o sistema detecta um reenvio antes de agir. Isso é idempotência no nível de webhook — obrigatória pela Res. BCB 195/2022. Sem isso, um reenvio pode liberar um pedido duas vezes ou dar baixa dupla em uma fatura.
Questão 14
Sobre o txid na API Pix dinâmica, qual afirmação está CORRETA?
O txid tem 26–35 chars, unicidade por CPF/CNPJ do recebedor, e fica apenas no payload JSON servido pela URL — nunca no BR Code. Pode ser gerado pelo recebedor ou delegado ao PSP (só em Cob).
Questão 15
Qual status de uma cobrança Pix (Cob/CobV) é final e irreversível — impedindo novos pagamentos?
CONCLUIDA é o único status verdadeiramente final e irreversível — indica que o Pix foi confirmado. Uma vez CONCLUIDA, nenhum outro pagamento pode ser feito nessa cobrança. O txid é único por CPF/CNPJ.
Questão 16
A CobV (cobrança com vencimento) impõe qual restrição ao tipo de QR Code?
A CobV é o "boleto do Pix" e o QR é obrigatoriamente dinâmico. Isso porque dados como vencimento, juros, multa e desconto precisam ser calculados em tempo real no servidor, e não podem ser estáticos.
Questão 17
O Art. 3-A incluído na Res. BCB 85/2021 pela Res. 538/2025 exige MFA obrigatório para qual acesso?
O MFA é exigido para acesso administrativo ao ambiente SPI — consoles, APIs de gestão, painéis de monitoramento. Não basta usuário + senha. (Art. 3-A, I, a da Res. BCB 85/2021)
Questão 18
No payload JSON de um QR dinâmico, o campo "apresentacao" deve conter qual valor?
O campo "apresentacao" deve ser preenchido pelo servidor no momento de atender a requisição — não é a data de criação. Isso permite ao app pagador verificar expiração em tempo real.
Questão 19
O Token Bucket implementado no DICT tem como objetivo principal:
O Token Bucket previne varredura em massa do DICT. PF tem 100 fichas; PJ, 1.000. Consulta inválida consome 20 fichas. Receber um Pix reabastece. Isso torna inviável o mapeamento sistêmico de CPFs por chaves.
Questão 20
Sobre a idempotência na API Pix e no SPI, qual afirmação está CORRETA segundo a Res. BCB 195/2022?
Art. 17 IV da Res. BCB 195/2022: idempotência é obrigatória para todos. Na API Pix: mesmo txid → retorna cobrança existente (HTTP 200), não duplicata. No SPI: mesmo E2EID → processado uma vez; reenvio é descartado sem erro.
